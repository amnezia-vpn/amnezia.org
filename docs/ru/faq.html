  • Можно ли доверять AmneziaVPN?

    Да. AmneziaVPN является проектом с открытым исходным кодом и разворачивает на сервере приложения OpenVPN, ShadowSocks, WireGuard и/или Cloak по желанию пользователя. У пользователя есть возможность как оперативно развернуть, так и полностью удалить любой из установленных на сервер VPN сервисов, а также полностью очистить сервер от всех установленных Амнезией сервисов всего в один клик. Вопрос в другом - доверяете ли Вы серверу, который собираетесь использовать? ;)

  • Как именно происходит установка?

    На ваш VPS будет установлен Docker. С его помощью будет собран контейнер, в котором и будет произведена установка выбранной базы (OpenVPN и т.д.). Благодаря этому AmneziaVPN легко обновляет всю необходимую начинку на сервере и позволяет без проблем изменить конфигурацию вашего VPN. Также, данный способ позволяет не "засорять" хостовую систему вашего сервера.

  • Почему требуется сервер с KVM?

    AmneziaVPN использует Docker для изолированного развертывания вашего VPN сервера. Не с каждым типом виртуализации возможно использование Docker, однако KVM это точно позволяет. С другими типами виртуализации работа с Docker может оказаться невозможной как в силу ограничений определенного типа виртуализации, так и в силу конфигурации от поставщика VPS.

  • Можно ли использовать авторизацию по ключам для AmneziaVPN?

    Можно, поддерживается авторизация как по паролю, так и по ключам. Поддерживаемые типы OpenSSH ключей: RSA (2048, 3072, 4096) и ECDSA (nistp256)

  • Как при авторизации по ключу AmneziaVPN установит все, не зная при этом пароля учетной записи?

    На текущий момент подразумевается что в учетке, указанной для логина на сервер, не будет запрашиваться пароль для операций с sudo

  • У меня случилась ошибка при использовании AmneziaVPN, что делать?

    Связаться с разработчиками (через support@amnezia.org или в Telegram чате) предоставив им логи работы программы. Достать логи можно по следующим путям:

    На Windows:

    • - C:\Users\%USERNAME%\AppData\Roaming\AmneziaVPN.ORG\AmneziaVPN\log
    • - C:\ProgramData\AmneziaVPN\log

    На MacOS:

    • - /$HOME/Library/Application\ Support/AmneziaVPN.ORG/AmneziaVPN/
    • - /var/log/AmneziaVPN/AmneziaVPN-service.log

    На Android: Здесь логи пока что не пишутся, но в будущем версия с логами обязательно появится!

  • Зачем может понадобиться сменить DNS сервер в настройках и какой лучше выбрать?

    По умолчанию AmneziaVPN использует DNS сервера Cloudflare. В сети есть множество других сервисов, которые помимо скорости и стабильности предоставляют дополнительный функционал: альтернативные протоколы для использования, блокировку рекламных и/или вредоносных доменов, "семейный режим", а некоторые сервисы могут поддерживать дополнительные доменные зоны. Помимо всего прочего, возможно, у Вас уже есть предпочитаемый DNS сервис, который Вы хотели бы поставить. А еще у AmneziaVPN есть DNS-контейнер, благодаря которому будут доступны сайты из emerDNS

  • Для чего предназначена кнопка "Clear client cached profile"?

    По умолчанию AmneziaVPN кэширует данные (сертификаты и ключи) от настроенного VPN сервера для быстрого подключения напрямую к с серверу. Данная кнопка сбрасывает этот кэш, в результате будут сгенерированы новые сертификаты с ключами, которые посредством SSH подключения будут помещены на сервер и уже только потом произойдет подключение. Необходимо использовать при смене сервера (если установлено на нескольких VPS), при обновлении клиента или сервера. Может потребоваться при изменениях настроек контейнеров.
